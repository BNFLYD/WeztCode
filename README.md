WeztCode no es un ide si no mas bien un hibrido entre TUI y GUI de código abierto para programar junto con editores de codigo en terminal como nvim (posteriormente nano, Helix e Incluso Emacs) en terminales modernas como WezTerm (pero posteriormente Alacrity, Ghosty y Warp) basicamente es una GUI agnostica para complementar aplicaciones completas de terminal que ya tienen todo lo necesario para competir con un Ide pero les falta ese toque de flexibilidad que un usuario principiante suele extrañar de Ides tradicionales como VSC, Sublime text, Zed, etc basicamente implementar las funciones tipicas de esos editores en sintonia con la velocidad inmediata que ofrece un entorno de terminal mas ligero y eficiente.

Las funciones que ofrece son:
- Una ventana lateral desplegable completamente personalizable con modulos como:
	- Un explorador de archivos para desplazarte, modificar, eliminar o crear archivos y/o carpetas nuevos.
	- Un chat de IA con opencode para utilizar tu modelo/servicio favorito de IA o si queres modelos locales tambien.
	- La capacidad de crear subagentes con reglas especificas con pi agents.
	- Un visualizador grafico para funciones de UX UI
	- La capacidad de crear widgets de estado para hacerlo completamente a tu imagen y semejanza
Todo lo demás ya lo tiene de por si nvim y no hace falta reinventar la rueda realmente, solo es una opción mas, para poder tener un entorno de trabajo simple y minimalista pero lo suficientemente extensible para no perder el tiempo saltando de una herramienta a la otra.
